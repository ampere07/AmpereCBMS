# Ampere Cloud Business Management System (CBMS)\n\nA full-stack web application built with React/TypeScript frontend and Laravel backend for business management functionality with Philippine market integration.\n\n## Project Structure\n\n```\nAmpereCBMS/\n├── frontend/          # React/TypeScript application\n├── backend/           # Laravel API\n└── README.md          # This file\n```\n\n## Quick Start\n\n### Prerequisites\n- Node.js (v14 or higher)\n- PHP (v8.0 or higher)\n- Composer\n- MySQL or SQLite\n\n### 1. Backend Setup (Laravel)\n\n```bash\ncd backend\n\n# Install dependencies\ncomposer install\n\n# Copy environment file\ncp .env.example .env\n\n# Generate application key\nphp artisan key:generate\n\n# Configure database in .env file\n# Set DB_CONNECTION, DB_HOST, DB_PORT, DB_DATABASE, DB_USERNAME, DB_PASSWORD\n\n# Run migrations\nphp artisan migrate\n\n# Start the backend server\nnpm start\n# OR traditionally: php artisan serve --port=8000\n```\n\nThe backend will run on http://127.0.0.1:8000\n\n### 2. Frontend Setup (React)\n\n```bash\ncd frontend\n\n# Install dependencies\nnpm install\n\n# Start the development server\nnpm start\n```\n\nThe frontend will run on http://localhost:3000\n\n## Key Features\n\n### Form Submission\n- Philippine mobile number validation (11-digit format starting with \"09\")\n- Geographic data integration (Region → City → Barangay)\n- Document upload support\n- Real-time validation feedback\n\n### Backend API\n- RESTful API endpoints\n- File upload handling\n- Database integration with proper migrations\n- Error handling and logging\n\n## API Endpoints\n\n- `POST /api/application/store` - Submit new application\n- `GET /api/applications` - Get all applications (paginated)\n- `GET /api/applications/{id}` - Get specific application\n- `GET /api/debug` - Debug information\n- `POST /api/reset-table` - Reset application table (testing)\n- `GET /api/health` - Health check\n\n## Database Schema\n\nThe `application` table includes:\n- Contact information (email, mobile, names)\n- Address details (region, city, barangay)\n- Plan selection and preferences\n- Document file paths\n- Application status tracking\n\n## Configuration Notes\n\n### CORS Setup\nThe backend is configured to accept requests from localhost:3000 for development.\n\n### File Uploads\nDocument uploads are stored in `public/assets/documents/` directory.\n\n### Custom npm start Command\nThe backend uses a custom \"npm start\" command instead of traditional Laravel serving methods, running on port 8000.\n\n## Development Workflow\n\n1. **Frontend Development**: Make changes in the `frontend/` directory\n2. **Backend Development**: Make changes in the `backend/` directory\n3. **Database Changes**: Create migrations in `backend/database/migrations/`\n4. **API Testing**: Use the `/api/debug` endpoint for debugging\n\n## Geographic Data\n\nThe application includes Philippine geographic data:\n- 17 regions (NCR, CAR, R1-R13, BARMM)\n- Cities/municipalities for each region\n- Barangays for major cities\n\n## Troubleshooting\n\n### Common Issues\n\n1. **500 Internal Server Error on form submission**\n   - Check if migrations have run: `php artisan migrate`\n   - Verify database connection in `.env` file\n   - Check Laravel logs: `storage/logs/laravel.log`\n\n2. **CORS Issues**\n   - Verify CORS configuration in backend\n   - Ensure frontend is running on correct port (3000)\n\n3. **File Upload Issues**\n   - Check `public/assets/documents/` directory exists\n   - Verify file permissions\n\n### Logs\n- Backend logs: `backend/storage/logs/laravel.log`\n- Frontend logs: Browser developer console\n\n## Environment Variables\n\n### Backend (.env)\n```\nDB_CONNECTION=mysql\nDB_HOST=127.0.0.1\nDB_PORT=3306\nDB_DATABASE=ampere_cbms\nDB_USERNAME=your_username\nDB_PASSWORD=your_password\n\nAPP_URL=http://127.0.0.1:8000\n```\n\n### Frontend\nNo additional environment variables required for basic functionality.\n\n## Contributing\n\n1.
